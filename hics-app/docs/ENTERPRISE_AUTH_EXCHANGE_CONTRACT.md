# Enterprise Auth Exchange Contract

This document defines the backend contract for exchanging OIDC authorization codes for NyxHICSlab app sessions.

## Endpoint

- Method: `POST`
- Path: `/auth/exchange`
- Content-Type: `application/json`

## Request Body

```json
{
  "provider": "entra",
  "code": "AUTHORIZATION_CODE",
  "codeVerifier": "PKCE_CODE_VERIFIER",
  "redirectUri": "https://app.nyxhicslab.com/auth/callback"
}
```

Fields:
- `provider`: `entra` or `okta`
- `code`: OIDC authorization code from identity provider callback
- `codeVerifier`: PKCE verifier generated by frontend
- `redirectUri`: Must match the redirect URI configured in the identity provider app registration

## Success Response

- Status: `200 OK`

```json
{
  "accessToken": "jwt-access-token",
  "refreshToken": "opaque-or-jwt-refresh-token",
  "expiresIn": 3600,
  "email": "it.admin@healthsystem.com",
  "fullName": "Enterprise Admin",
  "organization": "Enterprise Health System"
}
```

Fields:
- `accessToken`: App API bearer token
- `refreshToken`: Optional refresh token
- `expiresIn`: Seconds until access token expiry
- `email`: Normalized primary email claim
- `fullName`: Display name from identity provider
- `organization`: Tenant display name

## Error Responses

- `400 Bad Request`: Missing fields or invalid redirect URI
- `401 Unauthorized`: Invalid code or PKCE mismatch
- `403 Forbidden`: User not authorized for tenant or application
- `429 Too Many Requests`: Rate limited
- `500 Internal Server Error`: Unexpected provider or token service error

## Security Requirements

- Validate redirect URI against allowlist per environment
- Validate PKCE verifier and state on callback path
- Verify issuer, audience, nonce, and signature of provider tokens
- Map user to tenant and role claims before issuing app token
- Log auth events with correlation id for audit

## Frontend Integration Notes

- Frontend route: `/auth/callback`
- On success, frontend stores session and navigates to `/dashboard`
- On `401` from API, frontend should clear session and redirect to `/login`
